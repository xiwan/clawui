# v0.5.0 设计文档 — 流式预渲染加速

> **日期**: 2026-03-26
> **状态**: 设计中
> **前置版本**: v0.4.26

---

## 1. 问题

用户从输入到看到实际内容，端到端延迟 6-17 秒。

```
用户输入 → 骨架屏(50ms) → [Agent 完整执行 5-15s] → a2ui_render → liteRender(1-2s) → 渲染
                            ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                            这段时间用户只能看骨架屏 + 进度文案
```

Agent 内部时间分布（典型 web_search 场景）：

| 阶段 | 耗时 | 说明 |
|------|------|------|
| 意图理解 | ~1s | LLM 读 prompt，决定调什么工具 |
| 工具执行 | ~2-5s | web_search / shell / 其他工具 |
| 组织回复 | ~1-3s | LLM 把工具结果整理成 rawData |
| 调 a2ui_render | ~0.1s | Agent 发起 tool call |
| liteRender | ~1-2s | Haiku 选模板 + 格式化数据 |

关键洞察：工具执行完成后，rawData 实质上已经可用了。但当前架构要等 Agent 走完"组织回复 → 调 a2ui_render"才能拿到，白白浪费 1-3 秒。

---

## 2. 方案：工具结果预渲染

### 2.1 核心思路

利用 `runEmbeddedPiAgent` 的 `onToolResult` 流式回调，在 Agent 调完工具的瞬间拦截结果，立即并行启动 liteRender，不等 Agent 后续处理。

```
之前:
|--骨架屏--|--------Agent 完整执行(5-15s)--------|--liteRender(1.5s)--|--渲染--|
                                                                      ↑ 用户看到内容

之后:
|--骨架屏--|--Agent 开始--|--工具执行--|                               |--渲染--|
                                      |--liteRender(并行 1.5s)--|
                                                                ↑ 用户看到内容
                                      |--Agent 后续(1-3s)--|--a2ui_render(跳过)--|
```

省掉的时间 = Agent 组织回复(1-3s) + 重复 liteRender(1-2s) ≈ **2-5 秒**。

### 2.2 为什么可行

`RunEmbeddedPiAgentParams`（OpenClaw `src/agents/pi-embedded-runner/run/params.ts`）提供：

```typescript
onToolResult?: (payload: { text?: string; mediaUrls?: string[] }) => void | Promise<void>;
onPartialReply?: (payload: { text?: string; mediaUrls?: string[] }) => void | Promise<void>;
onAgentEvent?: (evt: { stream: string; data: Record<string, unknown> }) => void;
```

`onToolResult` 在每个工具执行完成后立即触发，payload.text 就是工具的原始输出——这正是 liteRender 需要的 rawData。

## 3. 设计

### 3.1 预渲染状态

每次用户操作创建，Agent 结束后清理：

```typescript
interface PreRenderState {
  intent: string;          // 用户意图（从 action.context.text 提取）
  toolResults: string[];   // 收集到的工具结果
  preRendered: boolean;    // 是否已推送预渲染
  liteResult?: LiteRenderResult;  // 预渲染结果缓存
}
```

生命周期：`fallbackToAgent()` 创建 → `onToolResult` 触发预渲染 → `a2ui_render execute` 检查并复用 → Agent 结束后置 null。

### 3.2 sendToAgent 改造

在 `runEmbeddedPiAgent` 调用中注入 `onToolResult`：

```typescript
async function sendToAgent(message: string, preRender?: PreRenderState) {
  // ... 现有路由解析逻辑不变 ...

  await api.runtime.agent.runEmbeddedPiAgent({
    // ... 现有参数不变 ...

    onToolResult: preRender ? async (payload) => {
      if (preRender.preRendered || !payload.text) return;
      if (payload.text.length < 50) return;  // 太短的结果跳过

      preRender.toolResults.push(payload.text);
      preRender.preRendered = true;

      try {
        const raw = preRender.toolResults.join("\n");
        const lite = await liteRender(raw, preRender.intent, { region: "us-east-1" });
        preRender.liteResult = lite;

        // 立即推送到前端，替换骨架屏
        const rendered = executeRender({ template: lite.template, data: lite.data });
        stopProgressStages();
        pushToPreview(rendered.jsonl, { template: lite.template, title: lite.data?.title as string });
        pushDoneHeader(lite.data?.title as string || lite.template || "完成");
        reportTokenUsage(lite.inputTokens, lite.outputTokens);

        api.logger?.info?.(`ClawUI pre-render: ${lite.ms}ms, template=${lite.template}`);
      } catch (e: any) {
        api.logger?.warn?.(`ClawUI pre-render failed: ${e.message}`);
        // 失败不影响主流程
      }
    } : undefined,
  });

  // Agent 结束，清理预渲染状态
  if (preRender) preRender.preRendered = true;  // 防止迟到的回调
}
```

### 3.3 a2ui_render 去重

Agent 最终调 `a2ui_render` 时，如果预渲染已经推送了相同内容，跳过重复渲染：

```typescript
// a2ui_render execute 内部
if (preRenderState?.preRendered && preRenderState.liteResult && (p as any).rawData) {
  // 预渲染已推送，跳过重复 liteRender
  const lite = preRenderState.liteResult;
  api.logger?.info?.(`ClawUI skip duplicate liteRender, using pre-render: ${lite.template}`);
  preRenderState = null;
  return {
    content: [{ type: "text", text: "UI already rendered via streaming pre-render." }],
  };
}
// 否则正常执行（Agent 用了 explicit template，或预渲染失败）
```

### 3.4 fallbackToAgent 入口

```typescript
function fallbackToAgent(action: any) {
  const meta = surfaceMeta.get(action.surfaceId || "main");
  const userText = (action.context?.text as string) || "";

  // 创建预渲染状态
  preRenderState = {
    intent: userText,
    toolResults: [],
    preRendered: false,
  };

  pushSkeleton(meta || { title: userText || undefined });
  handleUserAction(/* ... 不变 ... */);
}
```

### 3.5 数据流全景

```
用户点击 submit
    │
    ▼
fallbackToAgent()
    ├── preRenderState = { intent, preRendered: false }
    ├── pushSkeleton()  ──→ SSE → 前端显示骨架屏
    └── handleUserAction() → sendToAgent(message, preRenderState)
                                │
                                ▼
                          runEmbeddedPiAgent({
                            prompt: message,
                            onToolResult: callback
                          })
                                │
                    ┌───────────┤
                    │           │
              Agent 调工具    Agent 继续思考
                    │           │
                    ▼           │
              onToolResult()    │
                    │           │
                    ▼           │
              liteRender()      │  ← 并行执行
                    │           │
                    ▼           │
              pushToPreview()   │  ← 用户看到内容！
                    │           │
                    │           ▼
                    │     Agent 调 a2ui_render
                    │           │
                    │           ▼
                    │     检查 preRenderState
                    │     preRendered=true → 跳过
                    │           │
                    └───────────┘
                          │
                          ▼
                    Agent 结束
                    preRenderState = null
```

## 4. 边界情况

| 场景 | 行为 | 风险 |
|------|------|------|
| Agent 不调工具（纯问答） | onToolResult 不触发，走正常 a2ui_render | 无影响 |
| Agent 调多个工具 | 只对第一个 ≥50 字符的结果预渲染 | 可能选错模板，但 Agent 最终结果会覆盖 |
| 预渲染模板 ≠ Agent 最终模板 | Agent 的 a2ui_render 正常执行，覆盖预渲染 | 用户看到一次内容切换（可接受） |
| 预渲染失败（liteRender 超时/报错） | 静默忽略，Agent 正常走 a2ui_render 兜底 | 无影响，退化为当前行为 |
| Agent 用 explicit template（非 rawData） | preRendered 检查不命中，正常执行 | 无影响 |
| 工具结果太短（< 50 字符） | 跳过预渲染 | 无影响 |
| 声明式路由命中 | 不经过 fallbackToAgent，不创建 preRenderState | 无影响 |
| RAG 命中 | 不经过 fallbackToAgent | 无影响 |
| subagent.run 成功（Gateway 上下文） | 方式1 不支持 onToolResult，无预渲染 | 退化为当前行为 |

## 7. 测试计划

### 7.1 单元测试

在 `test/server.test.ts` 中新增：

- `pre-render: onToolResult triggers liteRender` — mock onToolResult 回调，验证预渲染推送
- `pre-render: short tool result skipped` — 工具结果 < 50 字符时不触发预渲染
- `pre-render: a2ui_render skips when pre-rendered` — 预渲染后 a2ui_render 的 rawData 模式跳过重复渲染
- `pre-render: a2ui_render normal when explicit template` — Agent 用 explicit template 时正常执行

### 7.2 手动验证

按 OPERATIONS.md §4.3 方法，SSE 监听 + `/action` 提交：

```bash
# 1. 清除状态
curl -s -X POST http://10.0.1.79:19001/reset

# 2. 后台监听 SSE，观察消息时序
timeout 60 curl -s -N http://10.0.1.79:19001/events | while IFS= read -r line; do
  [[ "$line" == data:* ]] && echo "[$(date +%H:%M:%S.%3N)] $line" | head -c 200
  echo
done &

# 3. 提交测试
curl -s -X POST http://10.0.1.79:19001/action \
  -H 'Content-Type: application/json' \
  -d '{"name":"submit","surfaceId":"main","context":{"text":"搜索今天的科技新闻"}}'
```

观察要点：
1. 骨架屏 ~50ms 出现
2. 预渲染内容在工具返回后 ~1.5s 出现（而非等 Agent 全部结束）
3. 不出现重复渲染闪烁（a2ui_render 被跳过）
4. sidebar 的 latency 显示缩短

### 7.3 对比测试

同一查询跑两次，一次开启预渲染，一次关闭（通过配置开关），对比端到端延迟。

## 9. 后续优化（不在 v0.5.0 范围）

- **liteRender 结果缓存** — 相同 rawData hash 直接返回，省掉 liteRender 的 1.5s
- **意图分类前置** — 简单查询跳过主 Agent，直接调工具 + liteRender，端到端 < 3s
- **RAG 重新启用** — 命中的查询直接返回或 replay
- **多工具结果合并** — 收集所有工具结果后再预渲染，提高模板选择准确率
