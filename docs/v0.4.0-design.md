# v0.4.0 设计文档 — 模板驱动 UI 生成

> **日期**: 2026-03-25
> **状态**: 实施中

---

## 1. 目标

让 LLM 永远不需要输出 A2UI JSON。LLM 只做两件事：选模板、填数据。

```
之前: LLM → 手写 A2UI components JSON（200-600 tokens）→ 渲染
之后: LLM → { template, data }（50-150 tokens）→ 模板引擎编译 A2UI → 渲染
```

基于 token-benchmark 测试结果：

| 场景 | A2UI 直出 | 模板+数据 | 节省 |
|------|----------|----------|------|
| 简单确认框 | 188 tokens | 95 tokens | 49% |
| 数据表格 | 280 tokens | 180 tokens | 36% |
| 复杂仪表盘 | 626 tokens | 326 tokens | 48% |

## 2. 现状分析

### 2.1 现有模板（8 个）

| 模板 | 场景 | 频率 |
|------|------|------|
| text_display | 纯文本 | 中 |
| form | 动态表单 | 高 |
| confirmation | 确认框 | 高 |
| booking_form | 餐厅预订 | 低（太专用） |
| search_results | 搜索结果 | 中 |
| dashboard | 指标卡片 | 中 |
| settings | 设置面板 | 低 |
| accordion | 折叠面板 | 低 |

### 2.2 覆盖缺口

Agent 常见场景中，以下没有模板覆盖，LLM 被迫 fallback 到手写 A2UI：

- 通用数据表格（服务器列表、订单列表、日志等）
- 操作结果反馈（成功/失败/进行中）
- 键值对详情（订单详情、用户信息）
- 多卡片网格（指标概览、功能入口）

### 2.3 当前 tool description 问题

`a2ui_render` 的 tool description 有 ~400 tokens，全在教 LLM 怎么写 A2UI components。这既浪费 input tokens，又引导 LLM 走"手写"路线。应该改为引导 LLM 优先选模板。

## 3. 设计

### 3.1 新增 4 个模板

**data_table** — 通用数据表格

```json
{
  "template": "data_table",
  "data": {
    "title": "服务器状态",
    "columns": [
      { "key": "name", "label": "名称" },
      { "key": "status", "label": "状态" },
      { "key": "cpu", "label": "CPU" }
    ],
    "rows": [
      { "name": "web-01", "status": "运行中", "cpu": "45%" },
      { "name": "web-02", "status": "运行中", "cpu": "72%" }
    ]
  }
}
```

引擎实现：columns 生成表头 Row，rows 逐行生成 Row，每个 cell 是 Text。

**status_page** — 操作结果反馈

```json
{
  "template": "status_page",
  "data": {
    "status": "success",
    "title": "部署完成",
    "message": "v1.2.3 已成功部署到生产环境",
    "details": [
      { "label": "耗时", "value": "2m 30s" },
      { "label": "实例数", "value": "3" }
    ]
  }
}
```

引擎实现：StatusIndicator + Text + 可选的 details 键值对列表。

**detail** — 键值对详情

```json
{
  "template": "detail",
  "data": {
    "title": "订单详情",
    "fields": [
      { "label": "订单号", "value": "#20260325-001" },
      { "label": "金额", "value": "¥299.00" },
      { "label": "状态", "value": "已发货" }
    ]
  }
}
```

引擎实现：标题 + fields 逐行生成 Row（label Text + value Text）。

**multi_card** — 多卡片网格

```json
{
  "template": "multi_card",
  "data": {
    "title": "功能入口",
    "columns": 3,
    "cards": [
      { "title": "搜索", "content": "搜索网页", "icon": "🔍", "action": "search" },
      { "title": "日程", "content": "管理日程", "icon": "📅", "action": "calendar" },
      { "title": "设置", "content": "系统设置", "icon": "⚙️", "action": "settings" }
    ]
  }
}
```

引擎实现：cards 按 columns 数分行，每行一个 Row，每个 card 是 Card 组件。

### 3.2 组合模板

支持 LLM 输出模板数组，引擎依次编译并拼接到同一个 Surface：

```json
{
  "templates": [
    { "template": "dashboard", "data": { "title": "概览", "metrics": [...] } },
    { "template": "data_table", "data": { "title": "明细", "columns": [...], "rows": [...] } }
  ]
}
```

实现：`tool.ts` 的 `executeRender` 检测 `templates` 数组参数，循环调用 `render()`，合并 components（id 加前缀避免冲突），用 Column 包裹。

### 3.3 重写 tool description

从"教 LLM 写 A2UI"改为"教 LLM 选模板"：

```
Render interactive UI. Choose a template and provide data.

Available templates:
- confirmation: { title, message, confirmLabel?, cancelLabel? }
- form: { title?, fields: [{ name, label, type }], submitLabel? }
- data_table: { title?, columns: [{ key, label }], rows: [{ key: value }] }
- dashboard: { title?, metrics: [{ label, value, trend? }] }
- status_page: { status: "success|error|info|loading", title, message?, details?: [{ label, value }] }
- detail: { title?, fields: [{ label, value }] }
- multi_card: { title?, columns?: 2|3, cards: [{ title, content?, icon?, action? }] }
- search_results: { query?, results: [{ title, snippet }] }
- text_display: { title?, body }
- settings: { title?, settings: [{ name, label, type: "toggle"|"text" }] }
- accordion: { sections: [{ title, content }] }

For complex pages, use "templates" array to combine multiple templates.

Only use "components" for truly custom layouts not covered by any template.
```

~200 tokens，比现在的 ~400 tokens 少一半，且引导 LLM 优先选模板。

## 5. 不做的事

- **不做 DSL 编译器** — token 测试证明模板方案更优（节省 36-49% vs DSL 的 24-36%），且实现成本低得多
- **不改 RAG** — 时效性问题暂不解决，RAG 继续服务静态页面缓存
- **不做骨架屏** — 留给 v0.5.0
- **不删现有 components 模式** — 保留作为 fallback，但 tool description 不再主推
